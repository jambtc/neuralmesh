# Abstract

NeuralMesh is a decentralized computational protocol designed to transform idle consumer hardware into a global artificial intelligence infrastructure.

Unlike traditional cryptocurrencies that consume energy solving non-useful cryptographic puzzles, NeuralMesh introduces a new consensus and reward model based on the execution of useful computational tasks related to artificial intelligence, distributed inference, scientific computation, rendering, and decentralized services.

The protocol enables anyone with consumer hardware to participate in the network and earn rewards by contributing CPU, GPU, memory, storage, or bandwidth resources.

**Minimum hardware requirements vary by task type:**

- LLM inference: 16 GB RAM, modern CPU (2020+); GPU with ≥ 8 GB VRAM recommended
- Image generation: GPU with ≥ 4 GB VRAM
- OCR, Speech synthesis, STT: CPU-only, 8 GB RAM sufficient
- Bandwidth contribution: any device with stable internet connection

The result is a decentralized intelligence layer for the Internet.

# Core Principles

- Useful computation instead of wasteful hashing
- Consumer hardware accessibility (no datacenter monopoly)
- Anti-centralization architecture
- Distributed AI inference
- Decentralized economic incentives
- Human-centric participation

> **Note on verification:** PoUC verification is probabilistic, not cryptographically proven.
> Every task runs on 2+ independent workers; results are compared via output similarity and hidden benchmark tasks.
> This provides measurable statistical confidence, not zero-knowledge proofs.
> Cryptographic proof of AI inference (zkML) remains an open research problem as of 2026.

## Competitive Positioning

NeuralMesh enters a field with established players (Bittensor, Akash, io.net, Golem).
Its differentiation:

- **Consumer hardware focus** — targets nodes with 8–24 GB VRAM, not datacenters
- **OpenAI-compatible API** — zero-friction drop-in for developers already using ChatGPT/Claude APIs
- **Privacy-first sandboxing** — task payloads cannot exfiltrate data (Docker, no network access)
- **Edge AI** — low-latency inference closer to end users, not only centralized cloud
